title: "@intl-party/client"

---

The client package provides auto-generated, type-safe translations for your project. It is generated by running `intl-party generate` and gives you a clean API with full TypeScript support.

## How It Works

Run the CLI to generate the client from your translation files:

```bash
npx intl-party generate --types
```

This reads your `messages/` directory and generates TypeScript types and runtime utilities automatically.

## Type-Safe Keys

```typescript
import { TranslationKey, Locale } from "@intl-party/client";

// TypeScript validates translation keys
const key: TranslationKey = "common.welcome"; // valid
const locale: Locale = "en"; // only valid locales allowed
```

## Runtime Utilities

```typescript
import {
  defaultMessages,
  createTranslationFunction,
} from "@intl-party/client";

// Access all messages
const messages = defaultMessages;
console.log(messages.en.common.welcome); // "Welcome to IntlParty!"

// Create a typed translation function
const t = createTranslationFunction("en", defaultMessages);
const translated = t("common.welcome"); // "Welcome to IntlParty!"
```

## Runtime Module

Additional utilities for runtime introspection:

```typescript
import {
  getLocaleMessages,
  getAllMessages,
  validateTranslationKey,
  getAvailableLocales,
} from "@intl-party/client/runtime";

// Get messages for a specific locale
const enMessages = getLocaleMessages("en");

// Get all messages across locales
const allMessages = getAllMessages();

// Validate keys at runtime
const isValid = validateTranslationKey("common.welcome"); // true

// List available locales
const locales = getAvailableLocales(); // ["en", "es", "fr"]
```

## Keeping the Client Updated

### Watch Mode

During development, run the generator in watch mode so it stays in sync:

```bash
npx intl-party generate --types --watch
```

### Auto-Generation

When using `@intl-party/nextjs` with `createNextConfigWithIntl`, set `autoGenerate: true` to regenerate types during builds automatically.

## Generated File Location

Generated files are placed at:

- Type definitions: `node_modules/.intl-party/types`
- Client package: `node_modules/@intl-party/client/generated`
