# Changesets

Hello and welcome! This folder has been automatically generated by `@changesets/cli`, a build tool that works
with multi-package repos, or single-package repos to help you version and publish your code. You can
find the full documentation for it [in the readme](https://github.com/changesets/changesets)

We use [changesets](https://github.com/changesets/changesets) to manage versions and changelogs in this monorepo.

## Adding a changeset

When you make a change that should be released, run:

```bash
pnpm changeset
```

This will prompt you to:
1. Select which packages are affected
2. Choose the semver bump type (major/minor/patch)
3. Write a summary of the changes

The changeset file will be committed with your PR. When the PR is merged, the release workflow will automatically version and publish the affected packages.

## For maintainers

To version packages based on accumulated changesets:

```bash
pnpm version-packages
```

To publish:

```bash
pnpm release
```
